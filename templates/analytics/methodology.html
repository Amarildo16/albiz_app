Core
Overview

Albiz is an analytical platform for exploring Albanian business registry and public procurement data. It separates collection and analysis into two independent components.

albiz_collector

Scraper, ETL, feature materialization, profiling, and audit-oriented data pipeline.

albiz_app

Django analytics and dashboard layer with read-only access to the collector database.

The integration contract between the two components is the database. The dashboard reads materialized collector tables and does not manage collector schema.
Core
Data Sources
APP procurement exports

Public procurement records are used to identify winner companies and procurement activity patterns.

QKB registry lookup data

Business registry search and lookup data provide company-level registry attributes for joined subjects.

Experimental
QKB historical discovery

Historical extract and document discovery are exploratory in the current version and are not yet fully batch-extracted.

Secondary
OpenCorporates

OpenCorporates is treated only as secondary exploratory enrichment, not as an authoritative core financial source.

Pipeline
Data Pipeline
1. Raw fetch

Source files and responses are collected by the pipeline.

2. Raw storage

Content hashing supports reproducibility and change detection.

3. Normalization

Identifiers, dates, amounts, and labels are standardized for analysis.

4. Feature materialization

Company-level feature tables are built for APP, QKB, and joined records.

5. Profiling and audit

Row counts, joins, and coverage metrics are checked after pipeline runs.

6. Dashboard access

Django reads the materialized tables through the collector database alias.

Core join
Exact NIPT Matching

APP winners and QKB subjects are joined only by exact normalized NIPT/NUIS. No fuzzy matching is used for the core joined dataset.

  • Exact matching reduces false positive links between unrelated entities.
  • The joined dataset contains companies that appear in both APP procurement data and QKB registry data.
  • Records with missing, malformed, or inconsistent identifiers may remain unmatched.
Indicators
Risk Indicators

Risk indicators are computed analytical signals that help prioritize entities for further review. They are not legal findings or determinations.

Winner/Budget > 1 Extreme ratio Zero budget with winner value Young company at first procurement High procurement count High winner value Suspended rate Cancelled rate QKB flag
Exploratory
Visual Analytics

Visual Analytics summarizes distributions by legal form, city, registration year, procurement activity, winner value, winner/budget ratio bands, and risk indicator distribution. These views are exploratory results based on joined APP-QKB companies.

Limitations
Limitations
  • Public source completeness depends on source availability and publication formats.
  • Exact NIPT matching may miss records with missing or incorrect identifiers.
  • Some APP records have missing winner NIPT values.
  • QKB historical documents are not fully batch-extracted in the current version.
  • Scanned PDFs and OCR are not part of the core pipeline.
  • OpenCorporates is not used as an authoritative core financial source.
  • Risk indicators are heuristic and should be interpreted as analytical signals.
Future work
Future Work
  • QKB historical document link extraction.
  • Financial document parsing from XLSX where available.
  • OCR as optional future work for scanned documents.
  • More advanced statistical and ML models.
  • Exportable analytical reports.
  • Read-only user roles and deployment hardening.
