# Template Translation Coverage Checklist

This checklist lists the templates and pages that need translation coverage.

General rule:

- Translate user-interface and explanatory text.
- Do not translate raw dataset values, company names, NIPT values, source names, model names, metric keys, CSV technical column names, or database-derived values.

## Layout Templates

| Area | File path | Priority | Translate | Do not translate |
| --- | --- | --- | --- | --- |
| Base layout | `templates/layouts/base.html` | High | Page title defaults, common labels, static fallback text | Static paths, block names, script names |
| Sidebar | `templates/layouts/partials/sidebar.html` | High | Navigation labels, section headings, ML dropdown labels | URLs, app route names, source names APP/QKB/OpenCorporates |
| Topbar/header | `templates/layouts/partials/topbar.html` | High | App subtitle, language switch labels, tooltips | Brand name `Albiz`, icon class names |
| Footer | `templates/layouts/partials/footer.html` | Medium | Footer text and static labels | Dates generated by code unless display-only |
| Global modals | `templates/layouts/partials/modals.html` | Medium | Modal titles, close labels, alert placeholders | Modal IDs, JS selectors |

## Strings Requiring Extra Care

Use consistent Albanian translations for:

- Procurement-based performance proxy.
- Analytical risk indicators.
- Heuristic weak label.
- Secondary financial enrichment.
- Exact NIPT matching.
- Exploratory ML analysis.
- Benchmark Suite.
- Human review required.

Avoid wording that implies:

- confirmed wrongdoing,
- automated decisions,
- complete financial coverage,
- official verified financial statements from OpenCorporates,
- real-world validation of heuristic labels.
